# Платформа для ДЗ по ЕГЭ (Валера)

Статический сайт для домашних заданий по тестовой части ЕГЭ: задачи с дедлайнами,
ввод ответов, автопроверка (ответы хранятся хешем, подсмотреть в коде нельзя),
сбор статистики в Google-таблицу.

## Что где лежит
- `index.html` — список заданий.
- `assignment.html` — страница одного задания с вводом ответов.
- `data/assignments.json` — сами задания и задачи (здесь добавляем ДЗ).
- `js/config.js` — адрес Google-таблицы (endpoint).
- `apps_script.gs` — код для Google Apps Script (приёмник статистики).
- `img/` — картинки к задачам.

## Запуск: 3 шага

### 1. Опубликовать сайт на GitHub Pages
0. Перед публикацией удали `reset.html` — это тестовый инструмент, ученику он не нужен.
1. Создай репозиторий на GitHub (например `pelegrina-math`) и залей туда все файлы этой папки.
2. В репозитории: Settings → Pages → Source: ветка `main`, папка `/root` → Save.
3. Через минуту сайт будет по адресу `https://<логин>.github.io/pelegrina-math/`.
   Эту ссылку даёшь Валере.

### 2. Подключить сбор статистики (Google-таблица)
Открой `apps_script.gs` — там пошаговая инструкция в комментарии сверху.
Коротко: создать таблицу → Расширения → Apps Script → вставить код → развернуть
как веб-приложение (доступ «Все») → скопировать URL `…/exec`.

### 3. Вписать адрес таблицы в сайт
В `js/config.js` вставь полученный URL в поле `endpoint` и залей изменение на GitHub.
Пока endpoint пустой, автопроверка работает, но статистика не сохраняется.

## Как добавлять новое ДЗ
Проще всего: попроси Claude «добавь ДЗ …, номера …, вот задачи и ответы» — он
сгенерирует блок в `assignments.json` с посчитанными хешами.

Вручную структура одного задания:
```json
{
  "id": "dz2",
  "title": "ДЗ №2 — графики функций",
  "students": ["valera"],
  "task_numbers": [11],
  "deadline": "2026-07-28T21:00:00",
  "intro": "Короткое пояснение для ученика (необязательно).",
  "problems": [
    {
      "id": "dz2-p1",
      "task_number": 11,
      "statement": "Текст задачи. Формулы в долларах: $y = x^2 - 4x$.",
      "image": "grafik1.png",
      "answer_hash": "СЮДА_ХЕШ_ОТВЕТА"
    }
  ]
}
```
- `students` — кому адресовано ДЗ: список id учеников, например `["valera"]` или
  `["valera", "kristina"]`. Если поле убрать (или поставить `"all"`), задание видят все.
  Список учеников и распознавание имён на входе настраивается в `js/config.js`.
- Формулы пишутся в формате LaTeX между знаками `$…$` (рендерит MathJax).
- Картинки кладём в папку `img/`, в поле `image` пишем только имя файла.
- `answer_hash` — SHA-256 от канонического ответа (Claude посчитает; вручную
  канонизация: убрать пробелы, запятую заменить на точку, число привести к
  обычному виду, например `2,50` → `2.5`).

## Вход учителя (админ)
Войди на сайте под именем **Лесечка** (или Леся / Олеся / admin). Админ видит
задания ВСЕХ учеников с меткой, кому что адресовано, и может открыть любое ДЗ.
Проверки под админом в статистику не пишутся. Список админов настраивается в
`js/config.js` (поле `admins`). Правильные ответы на сайте не показываются даже
админу (они хранятся хешем) — ключ ответов веди отдельно (см. трекер).

Чтобы у админа была кнопка «Открыть статистику», впиши обычный адрес Google-таблицы
(…/edit) в `js/config.js` → `statsSheetUrl`.

## Статистика
Каждая попытка ученика падает строкой в таблицу: время, ученик, задание, номер
задачи, ответ, верно/нет, номер попытки, успел ли к дедлайну. Дедлайн жёстко не
блокирует ввод (это статика), но в таблице видно, когда реально решал.

Есть и локальная резервная копия попыток в браузере ученика
(`localStorage`, ключ `attempts_backup`) на случай, если таблица была недоступна.
